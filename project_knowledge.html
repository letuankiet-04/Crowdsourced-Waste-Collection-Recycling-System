2. Kiến Trúc & Cấu Hình (Architecture & Configuration)
Cấu Trúc Thư Mục (Package Structure)
  • config: Cấu hình hệ thống (Security, Swagger, Cloudinary).
  • controller: Xử lý HTTP requests, chia theo actor (admin, citizen, collector, enterprise).
  • service: Logic nghiệp vụ chính.
  • repository: Tương tác database (Spring Data JPA).
  • entity: JPA Entities mapping với database tables.
  • dto: Data Transfer Objects (Request/Response).
  • exception: Xử lý lỗi tập trung.
Security & JWT (SecurityConfig, JWTHelper)
  • Authentication: Stateless, sử dụng JWT.
  • Authorization: Role-based (ADMIN, CITIZEN, COLLECTOR, ENTERPRISE).
  • Token Signing: HMAC-SHA512 (HS512).
  • Token Validation: Kiểm tra chữ ký, thời hạn và danh sách token bị hủy (JtiDenylistValidator).
Exception Handling (GlobalHandleException)
  • Sử dụng @RestControllerAdvice để bắt lỗi toàn cục.
  • Trả về format chuẩn ApiResponse.
  • Xử lý các lỗi: AppException (lỗi nghiệp vụ), MethodArgumentNotValidException (validate input), AccessDeniedException.
3. Logic Validation trong Service (Service Validations)

Chi tiết các kiểm tra logic nghiệp vụ trong các class Service Implementation.

  • AdminAccountService: Kiểm tra tồn tại User/Admin. Không cho phép Admin tự khóa tài khoản chính mình. Kiểm tra trạng thái User trước khi update (tránh update trùng).
  • AdminComplaintService: Kiểm tra tồn tại Complaint. Validate các entity liên quan (Request, Collector) trước khi xử lý vi phạm.

  • Register: Kiểm tra email trùng lặp. Validate các trường bắt buộc (email, password, fullname).
  • Login: Kiểm tra tài khoản tồn tại, mật khẩu đúng (BCrypt), tài khoản có bị khóa không.
  • Token: Verify token signature và expiry time. Kiểm tra token có nằm trong blacklist (logout) không.

  • CollectorReportCreationService:
    • Kiểm tra tồn tại: Request, WasteReport, Category.
    • Quyền: User phải là Collector và sở hữu Request đó.
    • Trạng thái: Task phải ở trạng thái COLLECTED. Report chưa được tạo trước đó.
    • Input: Phải có ít nhất 1 ảnh, 1 loại rác. Tổng khối lượng phải khớp.
  • CollectorService:
    • Kiểm tra Request thuộc về Collector.
    • Chuyển trạng thái task (Accept -> On The Way -> Arrived -> Collected) phải đúng quy trình.
    • Không cho phép Collector tự set trạng thái mình thành SUSPEND.

  • EnterpriseAssignmentService:
    • Collector phải thuộc Enterprise và đang ONLINE.
    • Collector không được là người đã từ chối task này trước đó.
    • Request phải ở trạng thái ACCEPTED_ENTERPRISE.
  • EnterpriseRequestService:
    • Địa chỉ Waste Report phải nằm trong khu vực hoạt động của Enterprise (Service Area Check).
    • Report phải ở trạng thái PENDING hoặc ACCEPTED_ENTERPRISE.
  • EnterpriseVoucherService:
    • Tên Voucher phải duy nhất.
    • Điểm đổi và số lượng tồn kho phải không âm.

  • WasteReportService:
    • Chỉ cho phép update/delete khi Report ở trạng thái PENDING.
    • Chống Spam: Không cho tạo report trùng vị trí trong vòng 10 phút.
    • Validate tọa độ địa lý (Latitude/Longitude).
    • Chỉ cho phép khiếu nại (Complaint) với các Report đã hoàn thành (COLLECTED).
  • VoucherService:
    • Kiểm tra Citizen đủ điểm đổi không.
    • Kiểm tra Voucher còn hạn và còn số lượng không.

    // Check points
    int cost = voucher.getPointsRequired();
    if (lockedCitizen.getTotalPoints() < cost) throw new AppException(ErrorCode.INSUFFICIENT_POINTS);

    // Check stock
    if (voucher.getRemainingStock() != null && voucher.getRemainingStock() <= 0) {
        throw new AppException(ErrorCode.VOUCHER_OUT_OF_STOCK);

    lockedCitizen.setTotalPoints(lockedCitizen.getTotalPoints() - cost);
    citizenRepository.save(lockedCitizen);
    
    // Deduct stock
    voucher.setRemainingStock(voucher.getRemainingStock() - 1);
    voucherRepository.save(voucher);

6. Chuyên Sâu: JWT & Spring Security Architecture

Hệ thống sử dụng mô hình Stateless Authentication với JSON Web Token (JWT) và Spring Security OAuth2 Resource Server.

1. Đăng Nhập & Cấp Token (Issue)
  • User gửi email/password tới /api/auth/login.
  • AuthService kiểm tra password (BCrypt).
  • JWTHelper tạo token với thuật toán HS512 (HMAC-SHA512).
  • Payload chứa:
    • sub: email người dùng.
    • iss: "team2.com".
    • exp: thời gian hết hạn (ví dụ: 1 giờ).
    • jti: UUID duy nhất cho token (dùng để logout).
    • scope: Danh sách quyền (ví dụ: ROLE_CITIZEN CREATE_REPORT).
2. Xác Thực Request (Verify)
  • Client gửi request kèm Header: Authorization: Bearer <token>.
  • Spring Security Filter Chain chặn request.
  • JwtDecoder (Nimbus) thực hiện:
    • Kiểm tra chữ ký (Signature) bằng signerKey.
    • Kiểm tra thời gian hết hạn (Expiration).
    • Gọi JtiDenylistValidator để kiểm tra xem jti có nằm trong bảng InvalidatedToken (blacklist) không.
  • Nếu hợp lệ: Security Context được thiết lập với Authentication object chứa danh sách quyền (Authorities).
  • Nếu không hợp lệ: Ném ngoại lệ, JwtAuthenticationEntryPoint trả về lỗi 401.
@Configuration
@EnableWebSecurity
@EnableMethodSecurity // Cho phép dùng @PreAuthorize trên method
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity httpSecurity) {
        // 1. Cho phép các endpoint public (login, register, swagger)
        httpSecurity.authorizeHttpRequests(req -> req
            .requestMatchers(PUBLIC_ENDPOINTS).permitAll()
            .anyRequest().authenticated()); // Các request khác phải đăng nhập

        // 2. Cấu hình OAuth2 Resource Server để xử lý JWT
        httpSecurity.oauth2ResourceServer(oauth2 -> oauth2
            .jwt(jwt -> jwt
                .decoder(jwtDecoder()) // Custom decoder (check signature + blacklist)
                .jwtAuthenticationConverter(jwtAuthenticationConverter())) // Map scope -> Authorities
            .authenticationEntryPoint(new JwtAuthenticationEntryPoint())); // Custom lỗi 401

        // 3. Tắt CSRF (vì dùng Token stateless)
        httpSecurity.csrf(AbstractHttpConfigurer::disable);
        
        // 4. Stateless Session (Server không lưu session)
        httpSecurity.sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS));

        return httpSecurity.build();
    }
}
Thành PhầnVai Trò & Logic
JWTHelper Issue Token: Tạo chuỗi JWT signed bằng HS512.
Verify Token: Parse token, verify signature, check expiry.
JtiDenylistValidator Implement OAuth2TokenValidator<Jwt>.
Mỗi khi request đến, nó truy vấn DB (bảng InvalidatedToken) xem ID của token (jti) có bị blacklist chưa (do logout).
JwtAuthenticationEntryPoint Xử lý lỗi 401 Unauthorized.
Thay vì trả về trang HTML lỗi mặc định của Spring, nó trả về JSON format chuẩn của dự án: { "code": 1001, "message": "Unauthenticated" }.
JwtAuthenticationConverter Chuyển đổi claim scope trong JWT thành danh sách GrantedAuthority của Spring Security.
Ví dụ: Scope ROLE_ADMIN -> Authority ROLE_ADMIN.

🔒 Security & Atomic Operations

Sử dụng Pessimistic Locking để xử lý giao dịch điểm thưởng an toàn, tránh Race Condition.

Code: Pessimistic Locking (Voucher) 
// VoucherServiceImpl.java
@Transactional
public VoucherRedemptionResponse redeem(Integer voucherId, String citizenEmail) {
    // 1. Lock Voucher (SELECT ... FOR UPDATE)
    Voucher voucher = voucherRepository.findByIdForUpdate(voucherId)
            .orElseThrow(() -> new AppException(ErrorCode.VOUCHER_NOT_FOUND));

    // 2. Lock Citizen (Tránh 2 request trừ điểm cùng lúc)
    Citizen citizen = citizenRepository.findByIdForUpdate(citizenId).get();

    // 3. Kiểm tra & Trừ điểm (Atomic trong Transaction)
    if (citizen.getTotalPoints() < voucher.getPointsRequired()) {
        throw new AppException(ErrorCode.INSUFFICIENT_POINTS);
    }
    citizen.setTotalPoints(citizen.getTotalPoints() - voucher.getPointsRequired());
    
    // ... save ...
}

// CollectorReportCreationService.java
private Calculation calculateItems(...) {
    int totalPoints = 0;
    double totalWeight = 0;
    
    for (int i = 0; i < categoryIds.size(); i++) {
        WasteCategory cat = categoryRepository.findById(categoryIds.get(i)).get();
        double weight = quantities.get(i);
        
        // Công thức: Điểm = (Hệ số rác * Khối lượng) * Tỷ lệ xác thực
        int points = (int) (cat.getPointPerKg() * weight * verificationRate);
        
        totalPoints += points;
        totalWeight += weight;
    }
    return new Calculation(totalPoints, totalWeight, ...);
}

VNPT eKYC Tutorial (Spring Boot)

Tài liệu triển khai & sử dụng module eKYC trong project Crowdsourced Waste Collection & Recycling System

Base URL VNPT API https://api.idg.vnpt.vn
Khuyến nghị không gọi VNPT trực tiếp từ frontend vì sẽ lộ Token-id/Token-key/Access token. Frontend chỉ gọi backend.

1) Cấu hình

application.yml

Các biến cấu hình đã được thêm trong vnpt.ekyc.

vnpt:
  ekyc:
    base-url: ${VNPT_EKYC_BASE_URL:https://api.idg.vnpt.vn}
    access-token: ${VNPT_EKYC_ACCESS_TOKEN:}
    token-id: ${VNPT_EKYC_TOKEN_ID:}
    token-key: ${VNPT_EKYC_TOKEN_KEY:}
    mac-address: ${VNPT_EKYC_MAC_ADDRESS:TEST1}
    timeout-ms: ${VNPT_EKYC_TIMEOUT_MS:15000}
Không hardcode token lên repo. Nên set bằng biến môi trường: VNPT_EKYC_ACCESS_TOKEN, VNPT_EKYC_TOKEN_ID, VNPT_EKYC_TOKEN_KEY.

Headers VNPT

Authorization
Bearer <access_token>
Token-id
<token_id>
Token-key
<token_key>
mac-address
TEST1 (configurable)

Backend tự inject toàn bộ headers này khi gọi VNPT.

Lỗi 401 Unauthorized gần như luôn là do Access token hết hạn hoặc Token-id/Token-key không khớp với token hiện tại.

2) Kiến trúc & luồng dữ liệu

Mô hình tích hợp: frontend → backend → VNPT. Backend đóng vai trò proxy để bảo vệ secrets và chuẩn hoá response. 

Frontend (React/Swagger/Postman)
  |
  | 1) POST /api/auth/ekyc/flow (multipart front/back)
  v
Backend (Spring Boot)
  | 2) Upload lên VNPT /file-service/v1/addFile  -> hashFront/hashBack
  | 3) Call AI:
  |    - /ai/v1/classify/id
  |    - /ai/v1/card/liveness
  |    - /ai/v1/ocr/id/front
  |    - /ai/v1/ocr/id/back
  |    - /ai/v1/ocr/id (OCR full)
  | 4) Lưu DB (ekyc_sessions): raw JSON + flags + profile extracted
  v
Client nhận: ekycSessionId + profile + raw JSON (nếu cần)

Lưu ý thực tế VNPT: ocr full có thể fail 400 trong khi ocr front/back vẫn OK. Vì vậy backend hiện validate đăng ký dựa trên classify + liveness + ocr front/back.

3) Endpoint backend & request/response

Base path: /api/auth/ekyc

Endpoint Consumes Mục đích
POST /upload (query: enhance) multipart/form-data Upload ảnh lên VNPT, trả về hash
POST /classify application/json Nhận diện loại giấy tờ từ hash
POST /liveness application/json Kiểm tra giấy tờ thật/giả
POST /ocr/front application/json OCR mặt trước
POST /ocr/back application/json OCR mặt sau
POST /ocr/full application/json OCR full (front + back)
POST /flow (query: clientSession, enhance, ...) multipart/form-data Full flow: upload → classify → liveness → ocr front/back → ocr full, đồng thời lưu DB
GET /sessions/{id} - Lấy lại kết quả eKYC đã lưu theo ekycSessionId

Giải thích các tham số quan trọng

Tham số Dùng ở đâu Ý nghĩa
clientSession /flow, các API AI Session theo định dạng VNPT: ANDROID_model_os_Device_sdk_deviceId_timestamp (khuyến nghị đúng format để tránh lỗi khó đoán).
token Body VNPT (classify/ocr) Chuỗi định danh do client tự tạo (không dùng ký tự đặc biệt). Không phải JWT của hệ thống.
type OCR -1 auto; 5 hộ chiếu; 6 bằng lái; 7 CM quân đội. Với CCCD mới thường ra type 2/3 từ classify.
cropParam OCR full Tỉ lệ crop 0.14,0.3 (dấu chấm thập phân). Nếu OCR full hay fail, thử bỏ trống để VNPT tự crop.
enhance /upload, /flow Bật chuẩn hoá ảnh trước khi upload VNPT (resize/re-encode/sharpen nhẹ) để giảm một số lỗi quality.

Ví dụ request/response nhanh

Upload

Upload không cần clientSession. Response backend trả hash để dùng cho các bước AI. 

curl -X POST "http://localhost:8080/api/auth/ekyc/upload?enhance=true&title=front&description=ekyc_front" \
  -H "Accept: application/json" \
  -F "file=@/path/to/front.jpg"

Classify (hash → loại giấy tờ)

curl -X POST "http://localhost:8080/api/auth/ekyc/classify" \
  -H "Content-Type: application/json" \
  -d '{
    "img_card": "<hashFront>",
    "client_session": "ANDROID_nokia7.2_28_Device_1.0_08d2d8686ee5fa0e_1710000000",
    "token": "abc123"
  }'

4) Enhance (chuẩn hoá ảnh) là gì?

enhance=true bật preprocessing phía backend trước khi upload lên VNPT. Mục tiêu là chuẩn hoá ảnh để model AI dễ xử lý hơn.

  • Convert RGB nền trắng (tránh PNG alpha)
  • Resize xuống tối đa cạnh dài ~ 2200px
  • Sharpen nhẹ
  • Encode JPEG quality cao (~0.92)
Enhance không “cứu” được ảnh chụp quá lóa/nhòe/mất góc. Nó chỉ giúp giảm các lỗi do định dạng/size/encode.

5) Full flow: cách gọi nhanh

Full flow dùng 2 file: front + back. clientSession nên theo format VNPT khuyến nghị. 

curl -X POST "http://localhost:8080/api/auth/ekyc/flow?clientSession=ANDROID_nokia7.2_28_Device_1.0_08d2d8686ee5fa0e_1710000000&token=abc123&type=-1&validatePostcode=true&cropParam=0.14,0.3&enhance=true" \
  -H "Accept: application/json" \
  -F "front=@/path/to/front.jpg" \
  -F "back=@/path/to/back.jpg"
Response trả thêm ekycSessionId và profile (thông tin người dùng trích xuất từ OCR front/back).

5) Lưu DB: bảng ekyc_sessions

Kết quả eKYC được lưu vào bảng ekyc_sessions để truy vấn/audit và phục vụ đăng ký.

Nhóm Cột Mô tả
Định danh id, user_id, client_session id là UUID string; user_id set khi đăng ký thành công kèm ekycSessionId
Hash ảnh hash_front, hash_back Hash VNPT trả về sau upload
Raw JSON classify_raw, liveness_raw, ocr_front_raw, ocr_back_raw, ocr_full_raw Lưu nguyên JSON VNPT để đối soát
Trạng thái bước *_ok, *_code Ok/codes theo message VNPT
Profile đã tách id_number, full_name, birth_day, origin_location, recent_location, issue_date, issue_place, card_type... Backend tự extract từ OCR front/back để client lấy nhanh

Gợi ý query nhanh

-- Tìm theo số giấy tờ
SELECT id, status, full_name, id_number, birth_day, created_at
FROM ekyc_sessions
WHERE id_number = '012345678901'
ORDER BY created_at DESC;

-- Tìm session của user đã đăng ký
SELECT id, user_id, status, full_name, id_number, created_at
FROM ekyc_sessions
WHERE user_id = 123
ORDER BY created_at DESC;

6) Thông tin người dùng trích xuất từ CCCD

Backend trích xuất profile từ ocrFront.object + ocrBack.object, lưu vào các cột chuẩn hoá và trả về field profile.

Field Nguồn VNPT Ghi chú
profile.fullName ocrFront.object.name (fallback name_label) Dùng để prefill form đăng ký
profile.idNumber ocrFront.object.id Số CMND/CCCD
profile.birthDay ocrFront.object.birth_day Chuỗi ngày theo VNPT (ví dụ 26/12/1989)
profile.recentLocation ocrFront.object.recent_location Địa chỉ thường trú
profile.issueDate ocrBack.object.issue_date Ngày cấp (mặt sau)
profile.issuePlace ocrBack.object.issue_place Nơi cấp (mặt sau)

6) Lấy lại kết quả đã lưu

Dùng ekycSessionId lấy lại toàn bộ kết quả và profile.

curl "http://localhost:8080/api/auth/ekyc/sessions/<ekycSessionId>"

6.1) Lưu ảnh lên Cloudinary (front/back)

Ngoài hash VNPT, hệ thống cũng upload ảnh CCCD lên Cloudinary để phục vụ audit/đối soát. Link ảnh được lưu trong bảng ekyc_sessions và trả về cho client.

Cột DB Ý nghĩa
front_image_url, front_image_public_id Ảnh mặt trước đã lưu Cloudinary
back_image_url, back_image_public_id Ảnh mặt sau đã lưu Cloudinary
Ảnh CCCD là dữ liệu nhạy cảm. Cần cân nhắc phân quyền xem ảnh (chỉ admin/CSKH) và chính sách lưu trữ/xoá dữ liệu.

7) Gắn eKYC vào đăng ký

Đăng ký có thể gửi kèm ekycSessionId. Backend sẽ validate eKYC đạt yêu cầu (classify + liveness + ocr front/back OK) rồi link session vào user. 

curl -X POST "http://localhost:8080/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user1@gmail.com",
    "password": "P@ssw0rd!",
    "fullName": "NGUYEN VAN A",
    "phone": "0901234567",
    "location": "HCM",
    "ekycSessionId": "<ekycSessionId>"
  }'
Nếu muốn bắt buộc eKYC cho mọi đăng ký, cần đổi rule tại register để không cho phép ekycSessionId null.

8) Troubleshooting nhanh

  • 401 Unauthorized: Access token hết hạn hoặc Token-id/Token-key mismatch → lấy token mới từ VNPT portal.
  • ocr full (400) quality nhưng front/back OK: thử bỏ cropParam, thử set type theo classify, hoặc giữ OCR full là optional.
  • Ảnh quality: đảm bảo đủ 4 góc, không lóa, không nén lại qua app chat; thử enhance=true.
  • Kiểm tra nhanh: gọi riêng /upload → /classify → /ocr/front → /ocr/back để biết fail ở bước nào.
